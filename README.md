# Mini Knowledge Base

##  Présentation
Projet collaboratif visant à documenter les bonnes pratiques Git, GitHub et méthodes Agiles.

##  Objectifs
- Appliquer GitHub Flow
- Travailler avec Issues et Pull Requests
- Mettre en place une CI simple
- Publier avec GitHub Pages

##  Structure du projet
##  Workflow Git (GitHub Flow)

1. Création d’une Issue
2. Création d’une branche `feature/...`
3. Développement
4. Pull Request
5. Review
6. Merge dans `main`

## Gestion Agile

- Backlog géré via GitHub Issues
- Suivi via GitHub Projects (Kanban)
- Priorisation : MUST / SHOULD / COULD
