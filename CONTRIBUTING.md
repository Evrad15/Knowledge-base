# Guide de contribution

##  Objectif
Ce projet est collaboratif. Toute contribution doit respecter le workflow Git défini.

---

##  Workflow Git

1. Créer une Issue décrivant la tâche
2. Créer une branche :
   feature/nom-fonctionnalite
3. Développer la fonctionnalité
4. Faire des commits clairs et réguliers
5. Push la branche
6. Ouvrir une Pull Request vers main
7. Attendre la review avant merge

---

##  Règles de commit

- Messages clairs et explicites
- Format recommandé :

USXX: Description courte

Exemple :
US15: Add GitHub Actions for Markdown validation

---

##  Règles importantes

- Interdiction de push direct sur main
- Une branche par User Story
- Review obligatoire avant merge

---

##  Organisation Agile

- Backlog géré via GitHub Issues
- Suivi via GitHub Projects (Kanban)
